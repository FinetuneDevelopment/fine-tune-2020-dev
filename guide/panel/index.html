Panels and call-to-action

CTA Box

Classes

Adding .cta-box to any element will give it some padding and allow it to fill Bootstrap columns.

Markup

.cta-box should not be added directly onto column elements. It should be the first element nested inside columns.

Colour

.cta-box is most effective when it has a colour tint applied to it which contrasts with the surround page.

Box link

Hover over the panels below, in order to see the effect

Classes

Add the class box-link to an immediate child of a Bootstrap column.

Markup

The class box-link should be applied to an anchor element.

Child

The class box-link__child should be applied to another element inside box-link. Content can be placed inside box-link__child, including full grid layouts.

Usage

Box link is used to link to related articles, such as Insights or Case Studies.

Endorsement quotes

Pull quotes should be used sparing on pages. As they have padding, they should not be combined with Bootstrap column elements, and should be nested inside them instead.

The element can be styled with the addition of a quote-endorsement class (ideally to a blockquote element).

Quest call-to-action

Classes

Elements can be styled using a cta-quest class.

Markup

As Quests padding, they should not be combined with Bootstrap column elements, and should be nested inside them instead.

Height

Quests have a minimum height, but will expand taller, if required to do so by content.

Column behaviour

If Quests sit as the only child of a Bootstrap column, they will automatically take the full height of that column. This allows them to be put together into mosaic grids with other Quests, or similar elements.
